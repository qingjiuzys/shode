# Shode 介绍

## 什么是 Shode？

Shode 是一个革命性的 Shell 脚本运行时平台，旨在解决传统 Shell 脚本的固有痛点，为自动化脚本提供安全、高效、现代化的执行环境。

## 核心愿景

将运维脚本从手工作坊模式提升到现代工程学科，创建一个统一、安全、高性能、拥有丰富生态的 Shell 脚本执行和开发平台。

## 主要特性

### 完整的 Shell 特性支持
- ✅ **控制流语句**: If-Then-Else、For/While 循环、Break/Continue
- ✅ **管道和重定向**: 支持 `|`, `>`, `>>`, `<`, `2>&1`, `&>` 等所有标准操作符
- ✅ **变量系统**: 变量赋值、展开 (`$VAR`, `${VAR}`)、字符串拼接
- ✅ **函数系统**: 函数定义、调用、参数传递、作用域隔离
- ✅ **模块系统**: 模块导入/导出、package.json 支持
- ✅ **注解系统**: 支持 `@Annotation` 语法，类似 Spring 框架
- ✅ **执行模式**: 解释执行（标准库）、进程执行（外部命令）、混合模式
- ✅ **安全特性**: 命令黑名单、敏感文件保护、模式检测
- ✅ **性能优化**: 命令缓存、进程池、性能指标收集

📖 [查看完整 Shell 特性清单 →](../guides/shell-features)

### 极致安全
- 内置安全沙箱，严格控制命令权限
- 自动拦截危险操作和注入攻击
- 执行前安全重写，实现安全左移

### 卓越性能  
- 标准库函数直接内存访问，无进程生成开销
- 智能执行模式选择（解释执行 vs 进程执行）
- 命令缓存和性能优化

### 现代生态
- 完整的包管理系统
- 模块化系统，支持代码复用
- 丰富的标准库函数
- 内置 HTTP 服务器、数据库、缓存支持

### 开发体验
- 完整的命令行工具链
- REPL 交互式环境
- 丰富的调试和诊断功能

## 适用场景

- **内部工具脚本**: 团队内部的自动化脚本执行
- **开发环境**: 开发者的本地脚本执行环境  
- **CI/CD 管道**: 构建和部署自动化
- **系统管理**: 服务器管理和运维任务
- **原型验证**: 技术验证和概念证明项目

## 快速体验

```bash
# 执行单条命令
shode exec "upper hello world"

# 运行脚本
echo "println('Hello, Shode!')" | shode run -
```
